## PROMPT 1 — R: Review Client Health Dashboard

**Purpose:** Generate a comprehensive 7-dimension health score analysis for a specific client.

**Best for:** Monthly check-ins, quarterly reviews, at-risk intervention.

```
You are a Client Success Strategist for an AI automation agency. Generate a detailed Client Health Dashboard for the following client.

CLIENT PROFILE:
- Company: [Client Name]
- Industry: [Industry]
- Monthly Retainer: $[amount]
- Engagement Length: [X] months
- Services Provided: [list services]
- Primary Contacts: [names and roles]

HEALTH DIMENSIONS TO SCORE (1-10):
1. Engagement Score — Meeting attendance, email response rate, Slack activity
2. Satisfaction Score — Recent feedback, NPS, complaint history
3. Value Perception — Do they articulate ROI? Case study testimonials?
4. Product/Service Adoption — Are they using what we built? Feature utilization %
5. Relationship Depth — Number of champions, executive access
6. Financial Health — Payment history, contract duration remaining, upsell potential
7. Competitive Risk — Have they mentioned competitors? RFP activity?

For each dimension, provide:
- Current score (1-10) with rationale
- Trend (↑ stable ↓) over last 3 months
- Key data points supporting your assessment
- Recommended action to improve

Then calculate:
- Overall Health Score (average of 7 dimensions)
- Health Tier: Green (8-10) / Yellow (5-7) / Red (1-4)
- Churn Probability: Low / Medium / High
- Expansion Readiness: Ready / Needs Nurturing / At Risk

Finally, provide a 3-paragraph executive summary of client health with specific action items for the next 30 days.
```

## PROMPT 2 — E: Evaluate Client NPS & Satisfaction Drivers

**Purpose:** Analyze satisfaction surveys, feedback, and sentiment to identify key drivers of client happiness (or dissatisfaction).

**Best for:** Post-QBR analysis, before renewal conversations, churn intervention.

```
You are a Customer Insights Analyst for an AI automation agency. Analyze the following client feedback data and identify key satisfaction drivers and improvement opportunities.

CLIENT: [Client Name]
INDUSTRY: [Industry]
TENURE: [X] months

RECENT FEEDBACK DATA:
[Paste survey responses, NPS scores, CSAT ratings, support tickets, verbatim comments, call transcripts, or any feedback touchpoints]

TASKS:

1. SENTIMENT ANALYSIS
- Classify each feedback item as Positive / Neutral / Negative
- Identify emotional tone (frustrated, delighted, confused, indifferent)
- Flag urgent issues requiring immediate attention

2. THEME CLUSTERING
- Group feedback into themes (e.g., response time, quality of work, pricing, communication, technical issues)
- Rank themes by frequency and severity
- Identify which themes the client mentions most vs. what they don't mention

3. KEY DRIVERS OF SATISFACTION
- What specifically does the client praise? (Be specific — quotes)
- What outcomes do they attribute value to?
- What would make them a promoter vs. passive?

4. KEY DRIVERS OF DETRACTION
- What are their top 3 frustrations?
- What expectations aren't being met?
- What competitors are they comparing us to?

5. RECOMMENDATIONS
- 3 quick wins (can implement in < 1 week)
- 3 medium-term improvements (1-4 weeks)
- 1 strategic initiative (1-3 months)
- A draft email or message addressing their top concern

OUTPUT FORMAT: Structured report with executive summary, detailed analysis, and action plan.
```

## PROMPT 3 — T: Track Engagement Metrics & Usage Trends

**Purpose:** Analyze engagement data across channels and identify usage patterns, drop-offs, and adoption gaps.

**Best for:** Monthly reporting, identifying under-engagement, proving value.

```
You are an Engagement Analytics Specialist for an AI automation agency. Analyze the following engagement data and provide actionable insights.

CLIENT: [Client Name]
PERIOD: [Date Range — last 30/60/90 days]

ENGAGEMENT DATA:
[Paste your data — meeting attendance rates, email open/click rates, Slack message volume, tool usage stats, dashboard logins, ticket volume, feature adoption rates, etc.]

ANALYSIS TASKS:

1. ENGAGEMENT TRENDS
- Week-over-week and month-over-month trends for each metric
- Identify spikes, dips, and patterns (seasonal, day-of-week, time-of-day)
- Benchmark against similar clients in your portfolio

2. ADOPTION SCORING
- For each service/product feature, rate adoption: High / Medium / Low / Not Used
- Identify features with high potential but low adoption
- Recommend a 3-step adoption campaign for each underutilized feature

3. DROP-OFF ANALYSIS
- Where in the client journey does engagement decline?
- What's the leading indicator of disengagement?
- Are there specific team members/departments that are disengaged vs. engaged?

4. VALUE CORRELATION
- Which engagement metrics correlate strongest with renewals? Upsells? Referrals?
- What's the minimum engagement threshold for a healthy client?
- Create a "green zone" target range for each key metric

5. INTERVENTION PLAN
- For any metrics in the "red zone," create a 14-day re-engagement plan
- Include specific outreach templates and cadence
- Define what "recovered engagement" looks like and how to measure it

OUTPUT: Engagement Health Score (1-100), trend indicators, and prioritized action plan.
```

- [ ] Increased support tickets (from [X] to [Y] per month)
- [ ] Negative feedback or tone shift
- [ ] Budget/cost conversations initiated
- [ ] New leadership or organizational changes
- [ ] Competitor mention or RFP activity
- [ ] Late payments or billing inquiries
- [ ] Reduced scope discussion
- [ ] New point of contact assigned

ADDITIONAL CONTEXT:
[Any other observations, conversations, or data points]

ANALYSIS:

1. CHURN RISK ASSESSMENT
   - Overall churn probability: Low / Medium / High / Critical

   - Secondary factors compounding the risk
   - Timeline estimate: When would they likely churn if nothing changes?

   - Watchlist indicators
   - Proactive check-in schedule
   - Preventative measures

## PROMPT 11 — E: Early Warning Alert System Configuration

**Purpose:** Set up automated triggers and alert rules that detect negative trends before they become churn risks.

**Best for:** Proactive account management, scaling client success operations.

```
You are a Retention Automation Engineer for an AI automation agency. Design an Early Warning Alert System.

AGENCY: [Agency Name]
CLIENTS: [X] active accounts
TOOLS USED: [CRM, communication tools, analytics platforms — list what you use]

SYSTEM DESIGN:

1. TRIGGER RULES (define 10-15)
   For each trigger, specify:
   - Event/Condition (what happens that should trigger an alert)
   - Data Source (where to monitor)
   - Threshold (at what point does it become actionable?)
   - Severity (Low/Medium/High/Critical)
   - Alert Recipient (CSM, Account Manager, Director)

   Example triggers:
   - No login for 14+ days → Medium → CSM
   - Support ticket volume up 50% MoM → High → CSM + Ops
   - Negative sentiment detected in email → Critical → CSM + Director
   - Late payment → High → Finance + CSM
   - Decision-maker leaves company → Critical → Account Manager
   - Feature usage drops 30%+ → Medium → CSM
   - Meeting rescheduled 3+ times → Low → CSM
   - Scope reduction request → High → Account Manager
   - Competitor mentioned in conversation → Medium → Account Manager
   - Contract renewal within 60 days → Low → CSM + Account Manager
   - NPS drops below 7 → High → Director
   - Executive outreach needed (no C-suite meeting in 90 days) → Medium → Account Manager

2. ALERT CHANNELS
   - Email notifications
   - Slack/Teams integration
   - CRM task creation
   - Dashboard alerts
   - SMS for critical alerts

3. ESCALATION PATHS
   - Level 1: CSM handles within 48 hours
   - Level 2: Account Manager + CSM Director within 24 hours
   - Level 3: CEO/Partner intervention within 4 hours

4. AUTOMATED RESPONSES
   - For Low severity: auto-send re-engagement email
   - For Medium severity: create CRM task + send email + notify CSM
   - For High severity: create urgent task + notify all levels + pull health report
   - For Critical: page entire leadership team + prepare retention package

5. MONTHLY AUDIT
   - Review all triggered alerts
   - False positive rate analysis
   - Threshold adjustment recommendations
   - New trigger rules to add

OUTPUT: Complete alert system configuration with trigger rules, escalation paths, and automation workflows.
```
